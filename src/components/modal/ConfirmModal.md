# ConfirmModal コンポーネント仕様書

## 実装目的

リソースの更新や削除操作を行う際に、ユーザーに確認を求めるためのモーダルコンポーネントを提供します。

## 実装先

`src/components/modal/ConfirmModal.tsx`

## 画面表示に関する要件

- モーダルは画面中央に表示され、背景は半透明・薄いグレーのオーバーレイで覆われる。
- モーダル内には以下の要素を含む：
  - タイトル（例：確認メッセージ）
  - 説明文（例：操作内容の詳細）
  - 操作に関する合言葉を入力するためのテキストフィールド（例：delete）
    - プレースホルダーとして合言葉を表示する。
  - 「確認」ボタン
  - 「キャンセル」ボタン
- 「確認」ボタンは主要なアクションとして強調表示される。
- 「キャンセル」ボタンは二次的なアクションとして表示される。

## propsに関する要件

- `isOpen: boolean` - モーダルの表示・非表示を制御するフラグ。
- `title: string` - モーダルのタイトルテキスト。
- `message: string` - モーダル内の説明文テキスト。
- `passphrase: string` - ユーザーが入力すべき合言葉。
- `onConfirm: () => void` - ユーザーが「確認」ボタンをクリックした際に呼び出されるコールバック関数。
- `onCancel: () => void` - ユーザーが「キャンセル」ボタンをクリックした際に呼び出されるコールバック関数。

## 動作に関する要件

- ユーザーが「確認」ボタンをクリックした場合、入力された合言葉が正しいか検証する。
  - 合言葉が正しい場合、指定されたコールバック関数を実行し、モーダルを閉じる。
  - 合言葉が間違っている場合、エラーメッセージを表示し、モーダルは閉じない。
- ユーザーが「キャンセル」ボタンをクリックした場合、モーダルを閉じる。
- モーダルは外部からのプロパティで表示・非表示を制御できる。
- モーダルが表示されるとき、フォーカスは自動的にテキストフィールドに移動する。

## スタイルに関する要件

- モーダルの背景は白色で、角に丸みを持たせる。
- テキストフィールドとボタンには適切なマージンとパディングを設定する。
- エラーメッセージは赤色で表示し、ユーザーの注意を引くようにする。
- レスポンシブデザインを考慮し、モバイルデバイスでも適切に表示されるようにする。

## アクセシビリティに関する要件

- モーダルはキーボード操作に対応し、Tabキーでフォーカスを移動できるようにする。
- モーダルが表示されるとき、背景のコンテンツへのアクセスを防止する。
- モーダルが閉じられたとき、フォーカスは元の位置に戻るようにする。
